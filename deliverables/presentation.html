RSNA Pneumonia Detection

Object detection on frontal chest radiographs (DICOM): localise pneumonia opacities with bounding boxes. Phase 1 scaffold, Liora MLE Cohort 6973.
TaskObject detection (bounding box)
ModalityChest X-ray (DICOM)
Records~26,684 patient studies
BaselineRetinaNet ResNet50-FPN + focal
AdvancedYOLOv8s + CLAHE + heavy aug
MetricmAP @ IoU 0.40-0.75

Motivation

Chest radiography is the highest-volume medical imaging exam. Pneumonia is among its most common indications. Inter-radiologist agreement on opacity extent is moderate, and demand frequently exceeds the supply of subspecialty chest radiologists. A model that localises pneumonia opacities (rather than only classifying the image) is more useful clinically because it supports an assistive workflow in which the radiologist remains the decision-maker.

Why bounding boxes, not just labels? A heatmap or box that aligns with the radiologist's reading is auditable and trustworthy. A scalar output is not.

Data

Source: RSNA Pneumonia Detection Challenge (Kaggle, 2018), a re-annotated subset of the NIH ChestX-ray dataset, with bounding-box ground truth produced by a panel of board-certified radiologists.

  • ~26,684 patient studies in stage_2_train_images/ (DICOM, ~1024 x 1024)
  • ~3,000 hidden test studies in stage_2_test_images/
  • One row per bounding box in stage_2_train_labels.csv (multi-row per patient)
  • Three-class metadata in stage_2_detailed_class_info.csv: Normal, No Lung Opacity / Not Normal, Lung Opacity
  • ~22-30 percent positive at the patient level
The middle class matters. "No Lung Opacity / Not Normal" patients have non-pneumonia abnormalities (cardiomegaly, effusion, atelectasis). A binary opacity-vs-rest model that ignores this stratum will silently underperform when deployed on a real clinical mix.

Total dataset size is ~3.7 GB. Per Liora Phase 1 rules, the dataset is documented but not downloaded; full kaggle competitions download -c rsna-pneumonia-detection-challenge instructions live in data/README.md.

Methods

Pre-processing pipeline

  1. Read DICOM with pydicom
  2. Min-max normalise pixel array, scale to uint8
  3. Resize to 512 (baseline) or 640 (advanced) pixels
  4. Apply CLAHE (Contrast Limited Adaptive Histogram Equalisation), clip 2.0, tile 8x8
  5. Replicate single channel to 3 channels for ImageNet-pretrained backbone

Train/val split

85/15 patient-level split, stratified on the per-patient maximum Target. Seed pinned to 42. Splits never cross patients.

Evaluation

Baseline: mAP@0.5 with eleven-point interpolated AP across validation images (Pascal-VOC style). Advanced: mAP@0.5, mAP@0.5:0.95 (Pascal), and the official RSNA challenge metric averaged over IoU 0.40 to 0.75 in 0.05 steps.

Baseline: RetinaNet ResNet50-FPN with focal loss

  • Backbone: ResNet50 + FPN, ImageNet-pretrained
  • Loss: focal loss (Lin 2017), bias init b = -4.6 for foreground prior 0.01
  • Optimiser: AdamW, lr 1e-4, weight decay 1e-4
  • Schedule: cosine, 6 epochs, batch size 4 at 512 x 512
  • Implementation: torchvision retinanet_resnet50_fpn_v2
Why RetinaNet for the baseline? Focal loss directly addresses the foreground-background imbalance that single-stage dense detectors face on RSNA. The implementation is short, auditable, and integrates with torchvision references.

Advanced: YOLOv8s with CLAHE + heavy augmentation

  • Architecture: Ultralytics YOLOv8s, anchor-free, decoupled heads
  • Pre-processing: DICOM-to-PNG with CLAHE applied during conversion
  • Resolution: 640 x 640, batch size 8, 30 epochs, patience 10
  • Augmentation: mosaic 1.0, mixup 0.15, translate 0.1, scale 0.5, fliplr 0.5
  • Anatomy guard: vertical flip disabled (heart is on the left)
  • Comparator: Faster R-CNN ResNet50-FPN-v2 (torchvision)
  • Out-of-scope: Detectron2 (compiled CUDA ops complicate Phase 1 install)

Discussion

Dataset bias is the elephant in the room

Zech et al. (2018) showed that a CXR pneumonia model exploited site-specific tokens (chest tubes, AP-vs-PA view, image-marker patterns) rather than disease features, and degraded sharply on radiographs from a different hospital. Our scaffold mitigates the most obvious traps (stratified split, preserved detailed-class metadata) but does not pretend to provide external validation.

Industrial vs medical CV (twin: project 10 Severstal)

Both projects are large-image, class-imbalanced, IoU-evaluated detection tasks. Industrial defects have crisp edges, deterministic illumination, and replicable ground truth. Medical opacities have soft edges, ambiguous extent, and probabilistic ground truth. The same backbone (ResNet50-FPN) is competitive in both, but the failure modes differ: industrial models fail on illumination shift, medical models fail on site/scanner shift.

Comparison with project 8 (COVID X-rays)

Project 8 uses the same modality with a classification framing. Both should cite the same Zech 2018 critique and the same Majkowska 2020 evaluation framework. The pre-processing (CLAHE) and backbone (ResNet50/DenseNet) are shared.

Limitations

  • Single dataset, single site distribution; no external validation
  • Hidden Kaggle test set is not used; reported numbers are from the public validation split only
  • No ensembling, no test-time augmentation, no multi-scale inference
  • Aggregate mAP can hide localisation issues; per-image error inspection is needed

Conclusion

Phase 1 produces a clean, runnable scaffold for pneumonia-opacity detection on the RSNA dataset. Two pipelines (RetinaNet baseline; YOLOv8 advanced + Faster R-CNN comparator) are documented end-to-end. Numerical results in this presentation are placeholders to be filled by Phase 2 by reading model-output JSON at build time, never by transcription.

Phase 2 readiness checklist. Accept Kaggle competition rules, run the documented kaggle CLI command, run python src/model_baseline.py then python src/model_advanced.py, then re-run the manuscript and presentation builds with the resulting metrics JSON files.
